## What This SDK Handles

- Validation of supported storage configuration.
- Video processing for `.mp4`, `.mov`, and `.mkv` files.
- HLS output generation with a master playlist and multiple variants.
- Optional Cloudinary publishing for generated HLS assets.

## Output Shape

Local processing returns a structured result:

```ts
{
  type: "video",
  variants: [
    {
      name: "hls",
      path: "output/<uuid>/master.m3u8",
    },
  ],
}
```

Cloudinary processing returns a single string:

```ts
"https://res.cloudinary.com/<cloud-name>/raw/upload/.../mediapipeline/<uuid>/master.m3u8";
```

## In Scope

- Local storage output.
- Cloudinary upload workflow.
- Exported config helpers and preset maps.

## Not In Scope

- Browser-side transcoding.
- Direct S3, GCS, Azure, or Cloudflare upload implementations.
- Image transformation pipelines.
